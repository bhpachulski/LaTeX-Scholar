\chapter{Avaliação do Processo para Desenvolvimento de Projetos de Pesquisa}

\section{Considerações Iniciais}

De acordo com Humphrey, a última etapa da definição de processos
se refere à validação da proposta. O autor propõe que seja
realizada uma execução simulada do processo usando dados de um
projeto que tenha sido completado recentemente. Em seguida, o
processo deve ser utilizado para o desenvolvimento de um pequeno
projeto ou de um protótipo.

Conforme apresentado na Seção \ref{aval1}, para a realização da
execução simulada do processo, foi escolhido o projeto SAFE
({\it Software Engineering Available for Everyone}), desenvolvido no
período de 2004 a 2006 por pesquisadores e estudantes de duas
universidades e por profissionais de uma empresa de
desenvolvimento de software. A execução do processo no
desenvolvimento de um projeto não foi realizada.

Para complementar a validação da proposta foi realizada uma
avaliação da estrutura do processo seguindo os critérios
estabelecidos por Horn \cite{hor90} apud Humphrey \cite{Hum95},
que se referem a princípios de mapeamento de informação. Os
resultados obtidos são apresentados na Seção X. Além disso, foi
realizada uma pesquisa com participantes do desenvolvimento de
projetos de pesquisa em que foi perguntado quais as principais
dificuldades que tiveram e como eles julgavam que um processo
poderia colaborar para minimizá-las. Os resultados são
apresentados na Seção x (emails + sbes).
